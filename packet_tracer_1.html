Packet Tracer with VLAN and Inter-VLAN Routing

Project Objective

Build a complex multi-router network that allows communication between hosts belonging to different VLANs (115 and 130), distributed across multiple switches and routers, using the Router-on-a-Stick technique and static inter-router routing.

General Topology

  • 4 routers (ROUTER ONE, TWO, THREE, FOUR) interconnected via point-to-point /30 links
  • 4 layer 2 switches (2960-24TT), each connected to a router and multiple PCs
  • VLAN 115 and VLAN 130 configured on each switch, with hosts assigned to each VLAN
  • Router subinterfaces configured for each VLAN (Router-on-a-Stick)
  • Static routing configured to ensure inter-VLAN and inter-site communication

Step-by-step Procedure

1. VLAN Creation
On each switch, the required VLANs were created: 
vlan 115
    name VLAN115
    vlan 130
    name VLAN130

2. Assigning Switch Ports to VLANs
Example: 
interface range fa0/1 - 2
    switchport mode access
    switchport access vlan 115
    interface range fa0/3 - 4
    switchport mode access
    switchport access vlan 130

3. Configuring Trunks between Switch and Router
interface fa0/5
    switchport mode trunk
    switchport trunk allowed vlan 115,130

4. Router Configuration: Router-on-a-Stick
Example from ROUTER ONE: 
interface g0/0.115
    encapsulation dot1Q 115
    ip address 20.20.20.1 255.255.255.0
    interface g0/0.130
    encapsulation dot1Q 130
    ip address 20.20.10.1 255.255.255.0
Same configuration on the other routers with different subnets:
  • ROUTER TWO: 30.30.30.1 for VLAN 115 and 30.30.10.1 for VLAN 130
  • ROUTER THREE: 50.50.50.1 and 50.50.10.1
  • ROUTER FOUR: 40.40.40.1 and 40.40.10.1

5. Host (PC) Configuration
Each PC manually configured with:
  • IP address
  • Subnet mask: 255.255.255.0
  • Default Gateway: IP of the router subinterface for the VLAN
Example for PC1 (VLAN 115, connected to ROUTER ONE): 
IP: 20.20.20.2
    Gateway: 20.20.20.1

6. Static Routing Configuration between Routers
Example ROUTER ONE: 
ip route 30.30.30.0 255.255.255.0 12.12.12.2
    ip route 30.30.10.0 255.255.255.0 12.12.12.2
    ip route 40.40.40.0 255.255.255.0 11.11.11.2
    ip route 40.40.10.0 255.255.255.0 11.11.11.2
    ip route 50.50.50.0 255.255.255.0 11.11.11.2
    ip route 50.50.10.0 255.255.255.0 11.11.11.2
ROUTER TWO: 
ip route 20.20.20.0 255.255.255.0 12.12.12.1
    ip route 20.20.10.0 255.255.255.0 12.12.12.1
    ip route 50.50.50.0 255.255.255.0 13.13.13.2
    ip route 50.50.10.0 255.255.255.0 13.13.13.2
    ip route 40.40.40.0 255.255.255.0 13.13.13.2
    ip route 40.40.10.0 255.255.255.0 13.13.13.2
...and so on for the other routers.

7. Verification and Troubleshooting
  • Trunk ports not properly enabled
  • Subinterfaces with duplicate or missing IPs
  • Incorrect gateways on PCs
  • VLANs not created on all switches
Diagnosis with: 
show vlan brief
    show ip interface brief
    show running-config
    ping
    traceroute

Each problem was solved by systematically checking all involved OSI layers (Layer 1-3).

Conclusion

The implemented network is a complete example of a multi-site infrastructure with inter-VLAN routing, simulated in Cisco Packet Tracer. The solution uses static routing and VLAN trunking to ensure scalability and logical traffic separation, maintaining centralized and stable routing.
